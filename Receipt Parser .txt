Receipt Parser 
Receipt Parser — система автоматического разбора кассовых чеков из фотографий с двухэтапным AI-пайплайном, REST API и Telegram-ботом, месендер MAX. Результат экспортируется в форматы XLSX/CSV, совместимые с 1С .
Назначение - использование бухгалтерами для легкого сканирования с личного телефона и отправкой файла либо на телефон, либо по почте либо напрямую в 1с. 

Как это работает
Система принимает изображение чека и прогоняет его через двухэтапный пайплайн :

Pass 1 — Извлечение (src/providers/openrouter.py): Vision-модель читает изображение и возвращает сырой JSON с данными чека

Pass 2 — Верификация (src/openrouter_client.py): вторая LLM-модель проверяет и уточняет названия товаров, даты, суммы

Нормализация (src/pipeline/normalize.py): приведение дат → YYYY-MM-DD, ИНН → только цифры 10/12 знаков, очистка хвостов НДС из названий, OCR-фиксы

Валидация (src/pipeline/validate.py): Pydantic-проверка схем

ResultBuilder (src/result_builder.py): сборка финального объекта ответа

Правила домена
Это критичные бизнес-правила, которые нельзя нарушать :

НДС не вычисляется — только читается с чека

Дата всегда приводится к YYYY-MM-DD или None

ИНН: только цифры, длина 10 или 12 (иначе пусто)

Названия товаров копируются максимально дословно

Позиции без НДС / 0% не получают искусственный vat_amount
