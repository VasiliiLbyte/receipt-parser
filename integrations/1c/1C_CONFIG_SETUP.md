# Настройка конфигурации 1С:Предприятие 8.3 с нуля

> Пошаговая инструкция + промты для Cursor
> Цель: создать минимальную конфигурацию для загрузки чеков из Receipt Parser API

---

## Предварительные требования

- Установлена 1С:Предприятие 8.3 (версия для разработчиков)
- Создана пустая информационная база (файловая, любая папка)
- Конфигуратор открыт в режиме "Конфигуратор"

---

## БЛОК А: Структура конфигурации

### А1. Справочник "Контрагенты"

**Путь в конфигураторе:**
Конфигурация → Справочники → ПКМ → Добавить

| Параметр | Значение |
|---|---|
| Имя | Контрагенты |
| Синоним | Контрагенты |
| Представление объекта | Наименование |

**Реквизиты (вкладка "Данные"):**

| Имя | Тип | Длина |
|---|---|---|
| ИНН | Строка | 12 |
| КПП | Строка | 9 |
| ЮридическийАдрес | Строка | 200 |

**Промт для Cursor:**
```
В конфигурации 1С:Предприятие 8.3 создай справочник "Контрагенты".
Реквизиты:
- ИНН (Строка, длина 12, не индексировать)
- КПП (Строка, длина 9)
- ЮридическийАдрес (Строка, длина 200)
Представление объекта — Наименование.
Покажи XML-описание объекта метаданных в формате EDT.
```

### А3. Перечисление "СтавкиНДС"

**Путь:** Конфигурация → Перечисления → ПКМ → Добавить

| Параметр | Значение |
|---|---|
| Имя | СтавкиНДС |
| Синоним | Ставки НДС |

**Значения (вкладка "Данные"):**

| Имя | Синоним |
|---|---|
| НДС20 | НДС 20% |
| НДС10 | НДС 10% |
| НДС0 | НДС 0% |
| БезНДС | Без НДС |

**Промт для Cursor:**
```
В конфигурации 1С:Предприятие 8.3 создай перечисление "СтавкиНДС"
со значениями: НДС20, НДС10, НДС0, БезНДС.
Синонимы: "НДС 20%", "НДС 10%", "НДС 0%", "Без НДС".
```

---

### А4. Документ "АвансовыйОтчет"

**Путь:** Конфигурация → Документы → ПКМ → Добавить

| Параметр | Значение |
|---|---|
| Имя | АвансовыйОтчет |
| Синоним | Авансовый отчет |

**Реквизиты шапки (вкладка "Данные"):**

| Имя | Тип | Уточнение |
|---|---|---|
| Контрагент | СправочникСсылка.Контрагенты | — |
| СуммаДокумента | Число | Длина 15, точность 2 |
| НомерЧека | Строка | Длина 20 |
| ДатаЧека | Дата | Только дата |
| СуммаНДС | Число | Длина 15, точность 2 |

**Табличная часть "Товары":**

Вкладка "Данные" → Табличные части → Добавить → Имя: Товары

Реквизиты табличной части:

| Имя | Тип | Уточнение |
|---|---|---|
| Номенклатура | СправочникСсылка.Номенклатура | — |
| Количество | Число | Длина 10, точность 3 |
| Цена | Число | Длина 15, точность 2 |
| Сумма | Число | Длина 15, точность 2 |
| СтавкаНДС | Строка | Длина 10 |
| СуммаНДС | Число | Длина 15, точность 2 |

**Промт для Cursor:**
```
В конфигурации 1С:Предприятие 8.3 создай документ "АвансовыйОтчет".

Реквизиты шапки:
- Контрагент (СправочникСсылка.Контрагенты)
- СуммаДокумента (Число, 15, 2)
- НомерЧека (Строка, 20)
- ДатаЧека (Дата, состав даты — Дата)
- СуммаНДС (Число, 15, 2)

Табличная часть "Товары" с реквизитами:
- Номенклатура (СправочникСсылка.Номенклатура)
- Количество (Число, 10, 3)
- Цена (Число, 15, 2)
- Сумма (Число, 15, 2)
- СтавкаНДС (Строка, 10)
- СуммаНДС (Число, 15, 2)

Покажи XML метаданных и код модуля объекта с проведением.
```

### А5. Роль "ОсновнаяРоль"

**Путь:** Конфигурация → Общие → Роли → ПКМ → Добавить

| Параметр | Значение |
|---|---|
| Имя | ОсновнаяРоль |
| Синоним | Основная роль |

После создания: открыть роль → нажать кнопку **"Установить все права"** → сохранить.

> ⚠️ Обязательно проставить галочки на все объекты: справочники Контрагенты и Номенклатура, перечисление СтавкиНДС, документ АвансовыйОтчет.

---

### А6. Пользователь "Бухгалтер"

**Путь (в режиме Предприятие):**
Администрирование → Пользователи → Создать

| Параметр | Значение |
|---|---|
| Имя | Бухгалтер |
| Полное имя | Бухгалтер |
| Пароль | (пустой) |
| Роли | ОсновнаяРоль |
| Основной интерфейс | Стандартный |

**В конфигураторе:**
Администрирование → Пользователи → Создать пользователя → указать имя "Бухгалтер", без пароля, роль ОсновнаяРоль.

---

## Порядок сборки конфигурации (последовательность важна)

1. Создать перечисление СтавкиНДС
2. Создать справочник Контрагенты
3. Создать справочник Номенклатура
4. Создать документ АвансовыйОтчет (ссылается на оба справочника)
5. Создать роль ОсновнаяРоль, дать права на все объекты
6. Обновить конфигурацию БД (F7 или меню Конфигурация → Обновить конфигурацию БД)
7. Запустить в режиме Предприятие, создать пользователя Бухгалтер

---

## Проверка конфигурации

После запуска в режиме Предприятия выполнить в Консоли кода:

```bsl
// Проверка наличия всех объектов метаданных
Сообщить("Контрагенты: " + Строка(Метаданные.Справочники.Содержит(Метаданные.Справочники.Контрагенты)));
Сообщить("Номенклатура: " + Строка(Метаданные.Справочники.Содержит(Метаданные.Справочники.Номенклатура)));
Сообщить("АвансовыйОтчет: " + Строка(Метаданные.Документы.Содержит(Метаданные.Документы.АвансовыйОтчет)));
Сообщить("СтавкиНДС: " + Строка(Метаданные.Перечисления.Содержит(Метаданные.Перечисления.СтавкиНДС)));
```

Ожидаемый вывод: все четыре строки `Истина`.
