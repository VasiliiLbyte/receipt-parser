# Быстрый старт: тестирование интеграции Receipt Parser + 1С

## Требования
- 1С:Предприятие 8.3 (любая редакция, в т.ч. для разработчиков)
- Receipt Parser API запущен на `http://localhost:8000`
- В боте добавлен хотя бы 1 чек (telegram `user_id` известен)

## Шаги

### 1. Подготовка базы 1С (10 мин)
1. Откройте пустую базу в режиме `Конфигуратор`.
2. Выполните инструкцию из `setup_test_config.bsl`:
   - создайте справочники `Контрагенты` и `Номенклатура`;
   - создайте документ `АвансовыйОтчет` с табличной частью `Товары`;
   - создайте перечисление `СтавкиНДС`;
   - создайте роль `ОсновнаяРоль` и пользователя `Тест`.
3. Обновите конфигурацию базы данных (F7) и запустите `1С:Предприятие`.

### 2. Первый тест — проверка соединения (2 мин)
1. В режиме `1С:Предприятие` откройте `Инструменты -> Консоль кода`.
2. Скопируйте и запустите скрипт из `test_load_receipt.bsl`.
3. При необходимости поменяйте:
   - `localhost`/порт API;
   - `UserID`;
   - `APIКлюч` (если включен `API_KEY`).
4. Убедитесь, что приходит ответ `HTTP 200` и JSON с чеками.

### 3. Тест внешней обработки (5 мин)
1. Откройте `Файл -> Открыть` и подключите внешнюю обработку.
2. Вставьте код из `ReceiptParserLoader.bsl` в модуль формы обработки (если требуется ручная сборка).
3. Проверьте реквизиты формы:
   - `СерверURL`
   - `APIКлюч`
   - `ПользовательID`
4. Нажмите кнопку загрузки и проверьте создание документов в тестовой базе.

### 4. Тест CommerceML XML (3 мин)
1. Получите XML из Receipt Parser API (`/export/xml`) или сохраните готовый файл.
2. В 1С выполните `Файл -> Открыть` и выберите XML-файл.
3. Проверьте, что данные чека корректно читаются и доступны для импорта в вашу конфигурацию.

## Связанные файлы
- `setup_test_config.bsl` — инициализация минимальной тестовой конфигурации
- `test_load_receipt.bsl` — базовый HTTP smoke-тест из 1С
- `ReceiptParserLoader.bsl` — внешняя обработка для загрузки чеков
