# End-to-End тест: от фото чека до документа в 1С

> Версия: 1.0 | Дата: 2026-03-24
> Стек: Python/FastAPI (порт 8001), aiogram, SQLite, 1С:Предприятие 8.3

---

## Подготовка

- [ ] API запущен на порту 8001 (`uvicorn main:app --port 8001`)
- [ ] Telegram-бот запущен (`python bots/telegram_bot.py`)
- [ ] 1С:Предприятие открыто с конфигурацией из Блока А (1C_CONFIG_SETUP.md)
- [ ] Конфигурация содержит: Контрагенты, Номенклатура, АвансовыйОтчет, СтавкиНДС
- [ ] Пользователь "Бухгалтер" создан и доступен

---

## Тест

### Шаг 1. Отправить чек через Telegram-бот

1. Открыть Telegram
2. Найти бота (имя бота из `.env` → `BOT_TOKEN`)
3. Отправить команду `/start`
4. Сфотографировать реальный кассовый чек и отправить боту
5. Дождаться ответного сообщения **"✅ Чек добавлен"** (или аналогичного)

> 💡 Если чек не распознаётся — попробуйте хорошо освещённое фото без теней.

---

### Шаг 2. Проверить API

Открыть браузер или curl:

```
http://localhost:8001/api/v1/receipts?user_id=769499969
```

**Ожидаемый результат:** JSON-массив с полем `receipts`, содержащим новый чек.

Пример структуры чека:
```json
{
  "id": 42,
  "date": "2024-10-19",
  "organization": "Интернет Решения ООО",
  "inn": "7704217370",
  "total": 3942.00,
  "items": [
    {
      "name": "Подписка Premium",
      "quantity": 1,
      "price": 3942.00,
      "amount": 3942.00,
      "vat_rate": "20%",
      "vat_amount": 657.00
    }
  ]
}
```

---

### Шаг 3. Загрузить чеки в 1С

1. Открыть 1С:Предприятие 8.3 под пользователем **Бухгалтер**
2. Меню **Файл → Открыть**
3. Выбрать файл `integrations/1c/ReceiptParserLoader.bsl`
4. В форме обработки заполнить:
   - **СерверURL**: `http://localhost:8001`
   - **ПользовательID**: `769499969`
   - **ДатаС / ДатаПо**: оставить пустыми (загрузить все)
5. Нажать кнопку **"Загрузить"**
6. Дождаться завершения (смотреть лог в поле сообщений)

**Ожидаемый лог:**
```
[10:32] Подключение к http://localhost:8001...
[10:32] Получено чеков: 1
[10:32] Создан контрагент: Интернет Решения ООО (ИНН 7704217370)
[10:32] Создан АвансовыйОтчет №000001 от 19.10.2024 | Интернет Решения ООО | 3942.00 руб.
[10:32] Готово. Загружено документов: 1
```

---

### Шаг 4. Проверить документ в 1С

1. Открыть меню **Документы → Авансовый отчет** (или через Журнал документов)
2. Найти созданный документ
3. Открыть и проверить:

| Поле | Ожидаемое значение |
|---|---|
| Номер | 000001 (автоматический) |
| НомерЧека | ID чека из API (например, "42") |
| ДатаЧека | Дата с чека |
| Контрагент | Интернет Решения ООО |
| СуммаДокумента | 3942.00 |
| Товары (ТЧ) | 1 строка с позицией чека |

4. Убедиться что документ **проведён** (статус "Проведён" или зелёная галочка)

---

### Шаг 5. Проверить справочники

**Справочник Контрагенты:**
- [ ] Существует запись "Интернет Решения ООО"
- [ ] Поле ИНН = 7704217370

**Справочник Номенклатура:**
- [ ] Существует запись с наименованием товара из чека

---

## Повторное тестирование (сброс данных)

1. В форме ReceiptParserLoader.bsl нажать кнопку **"Очистить"**
2. Подтвердить удаление
3. Запустить **Администрирование → Удаление помеченных объектов**
4. Повторить тест с шага 1

> ⚠️ Кнопка "Очистить" помечает документы на удаление, но не удаляет контрагентов и номенклатуру.

## Ожидаемый конечный результат

✅ Документ **АвансовыйОтчет** создан и проведён
✅ Данные совпадают с фото чека
✅ Контрагент найден или создан по ИНН
✅ Номенклатура найдена или создана по наименованию
✅ Суммы в табличной части совпадают с позициями чека

## Возможные ошибки и решения

| Ошибка | Причина | Решение |
|---|---|---|
| HTTP 404 / Connection refused | API не запущен | `uvicorn main:app --port 8001` |
| "Отсутствует документ АвансовыйОтчет" | Конфигурация не обновлена | Конфигуратор → F7 → Обновить БД |
| "Поле объекта не обнаружено" | Неверное имя реквизита | Проверить имена по 1C_CONFIG_SETUP.md |
| Документ не проводится | Нет прав у роли | Пересмотреть права ОсновнаяРоль |
| ДатаЧека = 01.01.0001 | Формат даты из API не распознан | Проверить поле `date` в JSON |
